## Base URLs

| Base | Purpose |
|---|---|
| `https://auth.riotgames.com` | SSO authentication and re-authentication |
| `https://entitlements.auth.riotgames.com` | Entitlements JWT exchange |
| `https://pd.{region}.a.pvp.net` | Player Data endpoints (shop, wallet, loadout, XP) |

Replace `{region}` with a two-letter region code: `na`, `eu`, `ap`, `kr`, `latam`, `br`.

## Authentication flow

Valstore uses a WebView-based SSO login. The flow is:

<Steps>
  <Step title="Open Riot login URL in WebView">
    `WebViewPage` loads the following URL in `flutter_inappwebview`:

    ```
    https://auth.riotgames.com/authorize
      ?redirect_uri=https%3A%2F%2Fplayvalorant.com%2Fopt_in
      &client_id=play-valorant-web-prod
      &response_type=token%20id_token
      &nonce=1
      &scope=account%20openid
    ```
  </Step>
  <Step title="Intercept the redirect">
    After a successful login Riot redirects to `https://playvalorant.com/opt_in#access_token=...`. The WebView intercepts this redirect and extracts the access token from the URI fragment.
  </Step>
  <Step title="Save cookies">
    `RiotService.saveCookies()` reads all cookies from `https://auth.riotgames.com/login` via `CookieManager.instance()` and persists them to `SharedPreferences` under the key `cookie`. These cookies are required for silent re-authentication.
  </Step>
  <Step title="Exchange for entitlements JWT">
    `RiotService.getEntitlements()` posts to `entitlements.auth.riotgames.com` with the access token to obtain the `entitlements_token`.
  </Step>
  <Step title="Decode user ID">
    `RiotService.getUserId()` decodes the `sub` claim from the access token JWT to obtain the player PUUID.
  </Step>
</Steps>

### Silent re-authentication

Background tasks and session refresh use `RiotService.reuathenticateUser()`:

```http
POST https://auth.riotgames.com/api/v1/authorization
Content-Type: application/json
User-Agent: RiotClient/{version} rso-auth (Windows; 10;;Professional, x64)
cookie: {saved-cookie-string}

{
  "client_id": "play-valorant-web-prod",
  "nonce": "1",
  "redirect_uri": "https://playvalorant.com/opt_in",
  "response_type": "token id_token",
  "response_mode": "query",
  "scope": "account openid"
}
```

The response contains a `response.parameters.uri` field. The new access token is extracted from the fragment of that URI.

## Required headers for PD endpoints

Every request to `pd.{region}.a.pvp.net` must include:

```http
Authorization: Bearer {accessToken}
X-Riot-Entitlements-JWT: {entitlementsToken}
X-Riot-ClientVersion: {riotClientVersion}
X-Riot-ClientPlatform: {base64EncodedPlatformJson}
```

`X-Riot-ClientPlatform` is the Base64 encoding of:

```json
{
  "platformType": "PC",
  "platformOS": "Windows",
  "platformOSVersion": "10.0.19042.1.256.64bit",
  "platformChipset": "Unknown"
}
```

`X-Riot-ClientVersion` is fetched at app start from `https://valorant-api.com/v1/version` (`data.riotClientVersion`). Falls back to `"08.09.00.2521387"` if the fetch fails during re-authentication.

## Endpoints

### POST /api/v1/authorization

**Re-authentication with cookies**

```http
POST https://auth.riotgames.com/api/v1/authorization
```

See [Silent re-authentication](#silent-re-authentication) above for request body and headers.

**Response** (relevant fields):

```json
{
  "type": "response",
  "response": {
    "mode": "fragment",
    "parameters": {
      "uri": "https://playvalorant.com/opt_in#access_token=eyJ...&token_type=Bearer&..."
    }
  }
}
```

Mapped to `Reauth` model (`lib/models/auth.dart`).

---

### POST /api/token/v1/

**Get entitlements token**

```http
POST https://entitlements.auth.riotgames.com/api/token/v1/
Content-Type: application/json
Authorization: Bearer {accessToken}
```

**Response** (relevant field):

```json
{ "entitlements_token": "eyJ..." }
```

### POST /store/v3/storefront/{uuid}

**Get full storefront (daily shop, bundles, night market)**

```http
POST https://pd.{region}.a.pvp.net/store/v3/storefront/{userId}
Authorization: Bearer {accessToken}
X-Riot-Entitlements-JWT: {entitlementsToken}
X-Riot-ClientVersion: {version}
X-Riot-ClientPlatform: {base64Platform}
Content-Type: application/json

{}
```

**Response** (top-level keys used by Valstore):

```json
{
  "FeaturedBundle": { "Bundles": [...], "BundleRemainingDurationInSeconds": 345600 },
  "SkinsPanelLayout": {
    "SingleItemOffers": ["uuid1", "uuid2", "uuid3", "uuid4"],
    "SingleItemStoreOffers": [
      { "OfferID": "uuid1", "Cost": { "85ad13f7-3d1b-5128-9eb2-7cd8ee0b5741": 1775 }, "Rewards": [...] }
    ],
    "SingleItemOffersRemainingDurationInSeconds": 72000
  },
  "BonusStore": {
    "BonusStoreOffers": [
      { "Offer": { "OfferID": "...", "Cost": {...} }, "DiscountPercent": 20 }
    ],
    "BonusStoreRemainingDurationInSeconds": 604800
  },
  "AccessoryStore": { ... }
}
```

Mapped to `Storefront` (`lib/models/storefront.dart`).

### GET /store/v2/offers/

**Get all purchasable offers**

```http
GET https://pd.{region}.a.pvp.net/store/v2/offers/
Authorization: Bearer {accessToken}
X-Riot-Entitlements-JWT: {entitlementsToken}
X-Riot-ClientVersion: {version}
X-Riot-ClientPlatform: {base64Platform}
```

Used by `RiotService.getLocalOffers()` to filter the full skin catalogue down to currently purchasable skins. Mapped to `LocalOffers`.


### PUT /name-service/v2/players

**Resolve player name and tag**

```http
PUT https://pd.{region}.a.pvp.net/name-service/v2/players
Content-Type: application/json
Authorization: Bearer {accessToken}
X-Riot-Entitlements-JWT: {entitlementsToken}
X-Riot-ClientVersion: {version}
X-Riot-ClientPlatform: {base64Platform}

["player-puuid"]
```

**Response**:

```json
[
  {
    "DisplayName": "PlayerName#TAG",
    "Subject": "player-puuid",
    "GameName": "PlayerName",
    "TagLine": "TAG"
  }
]
```
