Valstore is a Flutter application built on the **Provider** pattern. A single `ValstoreProvider` (extends `ChangeNotifier`) sits at the root of the widget tree and coordinates all state updates across screens.

## App entry point

`main.dart` performs three initialisation steps before the widget tree is mounted:

<Steps>
  <Step title="Firebase initialisation">
    `Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform)` is called with the generated options from `firebase_options.dart`. This must complete before any Firestore or Auth calls are made.
  </Step>
  <Step title="Workmanager initialisation">
    `Workmanager().initialize(callbackDispatcher, isInDebugMode: kDebugMode)` registers the top-level `callbackDispatcher` function (annotated `@pragma("vm:entry-point")`) as the background task entry point. This step is skipped on web.
  </Step>
  <Step title="NotificationService initialisation">
    `NotificationService.initialize()` sets up Android and iOS notification channels so that background tasks can fire notifications before the UI is mounted.
  </Step>
</Steps>

## Routes

| Route | Widget | Description |
|---|---|---|
| `/` | `HomeScreen` | Entry screen ã redirects to `/region` if no region is saved, otherwise shows login or home |
| `/region` | `RegionPage` | Region selection (saved to `SharedPreferences`) |
| `/login` | `WebViewPage` | In-app WebView SSO login via Riot's auth endpoint |
| `/store` | `ShopsPage` | Daily shop, featured bundles, night market |
| `/inventory` | `InventoryPage` | Player's owned skins |
| `/favorites` | `FavoritesPage` | Wishlisted skins from Firestore |

## Provider setup

`ValstoreProvider` is injected at the root `MaterialApp` via `ChangeNotifierProvider`:

```dart
ChangeNotifierProvider(
  create: (context) => ValstoreProvider(),
  builder: (context, child) {
    return MaterialApp(
      routes: routes,
      navigatorKey: navigatorKey,
      theme: themeData,
    );
  },
)
```

The global `navigatorKey` allows background-task code and services to push routes (e.g. redirect to `/region`) without a `BuildContext`.

## Background tasks

Valstore uses **Workmanager** to run periodic store checks while the app is closed. Three task names are registered:

<AccordionGroup>
  <Accordion title="ValStoreStoreRenewal">
    Calls `RiotService.recheckStore()`. Re-authenticates the user, fetches the current daily shop, and fires a local notification for any wishlisted skin that is in rotation.
  </Accordion>
  <Accordion title="NightMarketRenewal">
    Calls `RiotService.recheckNightmarket()`. Checks whether the Night Market is active and sends a one-time notification when it opens. A `didNotifyNM` flag in `SharedPreferences` prevents repeat notifications.
  </Accordion>
  <Accordion title="ValStoreBundleRenewal">
    Calls `RiotService.recheckBundle()`. Compares the current bundle ID against the last known bundle stored in `SharedPreferences`. Sends a notification with the bundle artwork when a new bundle is detected.
  </Accordion>
</AccordionGroup>

All three task callbacks re-initialise Firebase independently (in case the app process is cold-started by Workmanager) before calling the relevant `RiotService` method.

## Theme

The app uses **Material 3** with a dark colour scheme defined in `lib/theme.dart`. Key colours:

- Primary / accent: `#FF4655` (Valorant red)
- App bar / drawer background: `#16141A`
- Font: [Nunito](https://fonts.google.com/specimen/Nunito) via `google_fonts`
